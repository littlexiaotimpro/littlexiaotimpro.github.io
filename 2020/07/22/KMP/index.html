KMP - 字符串匹配算法

发表于 2020-07-22 | 分类于 notebook

1. 概述

KMP算法是一种改进的字符串匹配算法，由D.E.Knuth，J.H.Morris和V.R.Pratt提出的，因此人们称它为克努特—莫里斯—普拉特操作（简称KMP算法）。KMP算法的核心是利用匹配失败后的信息，尽量减少模式串与主串的匹配次数以达到快速匹配的目的。具体实现就是通过一个next()函数实现，函数本身包含了模式串的局部匹配信息。

作为必须了解的效率算法之一，在适用的场景使用合适的算法可以提升程序的执行效率，同时减少不必要的资源损耗 ……

2.1. 模式串前缀表

  • 什么是前缀表？
  • 为什么需要前缀表？
a. 什么是前缀表？

前缀表，也就概述中提到的 next() 函数所实现的功能得到的结果，用来表示，在匹配失败后模式串相对于原先位置需要移动到什么位置，而把模式串中每一个字符对应的移动位集合起来就构成了一个表，如下：

模式串字符组 a b c a b e
字符索引 0 1 2 3 4 5
最长公共前后缀 0 0 0 1 2 0
失配对应索引位 -1 0 0 0 1 2

最长公共前后缀的由来（前缀后缀不含字串本身）：

模式串子串 前缀 后缀 最长公共前后缀
a - - -
ab a b -
abc a, ab c, bc -
abca a, ab, abc a, ca, bca a
abcab a, ab, abc, abca b, ab, cab, bcab ab
abcabe a, ab, abc, abca, abcab e, be, abe, cabe, bcabe -

如上，很直观的就可以找到模式串各字串的最长公共前后缀。

b. 为什么需要前缀表？

反观暴力匹配算法，在对应字符不匹配时，模式串右移一位，出现许多重复比较的字符，秉着多滑动几位的思想尽可能减少匹配次数，前缀表的衍生规则，(参照，BM算法)：

  • 坏字符规则（从左向右）
    出现的不匹配字符认定为坏字符，开始向右移动，直到匹配到相同字符，若坏字符在模式串中不存在，则可以认为模式串可以滑动至索引为-1的位置，若存在多个坏字符，则以最右边的字符匹配为主，谨防移动过多漏了可能的结果。
  • 好后缀规则
    在仅有坏字符规则的前提下，匹配容易出现位移倒退的情况，比如T: aaaaaaaaaaa, P: baa。因此，结合好后缀规则，确定模式串合适的移动位置，实现效率最大化。

规则的选择标准，一般分别计算好后缀和坏字符规则往后滑动的位数，取大的，作为移动位数，因为两种规则可以独立使用，考虑资源损耗，优先选择好后缀规则。

KMP算法的前缀表是在BM算法的匹配规则上进行了优化，将规则的选择，改成了使用前缀表，使用中不需要纠结使用哪种规则。

KMP算法是在失配位置处，判断失配位置之前的模式串子串的最大公共前后缀，若存在，则移动模式串至前后缀匹配处，若不存在，则移动首字符至失配位置，继续判断。

3. 时间及空间复杂度

假定主串长度为n，模式串长度为m，那么next数组所占的空间为可以记为m。

空间复杂度为：O(m)。

对模式串进行遍历构建next数组，时间复杂度可以估算为m，然后对主串进行循环匹配，时间复杂度为n

因此算法的时间复杂度：O(m+n)。
