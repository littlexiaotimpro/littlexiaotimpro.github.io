冒泡排序

冒泡排序（Bubble Sort），特点是将待排序元素像气泡一样向上漂浮，实现升序或降序的目的。

实现逻辑是取当前待排序数组两两进行对比，若相邻元素比当前元素值小（或大）就交换他们的位置，重复这个过程，一次循环结束以后，最后一个元素就是最大值（或最小值）。

时间复杂度：
假设数组长度为 n，那么内层循环的元素比对时间复杂度可以近似为 O(n)，元素交换过程时间复杂度为 O(1)，因此总的时间复杂度为 n*O(n)*O(1) = O(n^2)。

需要注意，冒泡排序的性能与待排序序列的有序程度有关，越接近有序的序列，算法执行效率越高。

空间复杂度：
因为是原地排序，所以空间复杂度为 O(1)。

稳定性：
元素在排序过程中，如数组 [4,3,3']，升序后得到 [3,3',4]，其中 3 和 3’ 在排序过程中，相对位置并没有变化，所以是 稳定排序。

选择排序

选择排序（Selection sort），通过每次循环遍历确定当前的最大值或最小值，将他们替换到指定索引位实现排序。

实现逻辑是以当前遍历索引为标准，假定该元素为最小（或大）值，依次和之后的元素进行对比，若查找元素比当前元素值小（或大）就替换当前的目标索引，继续遍历，一次循环结束以后，替换当前索引和目标索引的元素值，此时，当前元素值就是最小值（或最大值），从下一个元素开始，重复此过程。

时间复杂度：
假设数组长度为 n，那么内层循环的元素比对时间复杂度最差可以近似为 O(n-1)，即每一个元素都发生了一次交换，因此元素交换过程时间复杂度最差为 O(n-1)，因此总的时间复杂度近似为 O(1+2+3+...+n-1) = O(n^2/2 - n/2) ~ O(n^2)。

需要注意，选择排序的性能与同样与待排序序列的有序程度有关，越接近有序的序列，算法执行效率越高。

空间复杂度：
因为是原地排序，所以空间复杂度为 O(1)。

稳定性：
元素在排序过程中，如数组 [4,3,3']，升序后得到 [3',3,4]，但在比较过程中，遍历需要走到最后，因此最后一个 3 和优先和 4 交换位置，最终结果里两个 3 发生了相对位置的交换，所以是 非稳定排序。

插入排序

插入排序（Insertion sort），就好比玩斗地主的时候，将新抓一张牌，插入已有序的牌堆的适当位置，因此插入排序主要分为两步：

  • 查找元素的插入位置
  • 将已排序的数组的插入位置之后的元素后移，再插入元素

时间复杂度：
假设数组长度为 n，那么内层循环的元素比对时间复杂度最差的情况为 O(n-1)，因此总的时间复杂度近似为 O(1+2+3+...+n-1) = O(n^2/2 - n/2) ~ O(n^2)。

需要注意，插入排序的性能与同样与待排序序列的有序程度有关，越接近有序的序列，算法执行效率越高，最优的情况就是比较最后一个元素，总的时间复杂度为 O(n-1)，最差的为 O(1+2+3+...+n-1) --> O(n^2)。

空间复杂度：
因为是原地排序，所以空间复杂度为 O(1)。

稳定性：
元素在排序过程中，如数组 [4,3,3']，3 和 4 比较后，移动至 4 之前，得到 [3,4,3']，3’ 和 4 比较后，在与 4 相邻的位置前直接插入，得到 [3,3',4]，两个 3 的相对位置没有发生变化，所以是 稳定排序。

希尔排序

希尔排序（Shell’s sort），是插入排序的一种又称“缩小增量排序”（Diminishing Increment Sort），是直接插入排序算法的一种更高效的改进版本。

排序主要通过分组的方式实现元素排序，按一定增量对数组进行分组，每一个分组都进行直接插入排序，直到增量缩减为 1 时，此时的数组就是已经排好序的序列。

时间复杂度：
希尔排序的时间复杂度优于直接插入排序，增量每次除 2 取整，总次数为 2^x = n => x = log2(n)，内层循环为插入排序的算法，时间复杂度为 O(n/2+n/4+... + n-1) ~ O(n)，因此时间复杂度为 O(n*log2(n))。

空间复杂度：
因为是原地排序，所以空间复杂度为 O(1)。

稳定性：
因为希尔排序是通过实现各分组排序来达到整个元素列有序，已知插入排序是稳定排序，也就是各分组的排序是稳定，但总体来说，可能存在元素发生相对位置变换，如：

快速排序

快速排序（Quicksort），又称分区交换排序（partition-exchange sort），是冒泡排序的一种改进算法，最早由东尼·霍尔提出。

实现逻辑选取一个值为标准（一般默认第一个），用个变量存储，然后，从右边开始找，直到第一个小于标准值的数，交换它们的位置，然后从左边开始找，直到第一个比标准值大的数，交换它们的位置，重复此操作，最终结果是 左边的序列 < 标准值 < 右边的序列，然后通过递归的思想，对左右子序列分别在进行上述操作，直到结束。

因为是双向遍历，所以结束条件就是，当右边指针小于等于左边指针时，结束递归。

时间复杂度：
在平均状况下，排序 n 个项目要 O(n*log(n)) 次比较。在最坏状况下则需要 O(n^2) 次比较，推导过程如下：

正常情况下，对待排序序列执行快速排序时，需要将原序列按标准值，划分成两个子序列，并且满足 左子序列 < 标准值 <= 右子序列 ，递归执行此操作，可以得到子序列的个数满足 2^x >= n 得，x >= log2(n)，排序的内层循环的次数为 O(n)，因此正常情况下的时间复杂度为 O(n*log2(n))。

需要注意，如果已经存在一个排好序的升序序列，在执行快速排序时，因为一次内层循环最多只能确定一个元素有序，执行时间为 O(n)，因此，为了确认序列顺序，需要递归整个序列，及需要执行 n 次内层循环，因此排序所需要的时间反而更多了，为 O(n^2)。

空间复杂度：
每一次递归都需要临时变量来存储标准值和临时变量，因此空间复杂度为 O(3*log2(n)) ~ O(log2(n))。

堆排序

了解堆排序前，需要知道什么是堆：

  • 完全二叉树
    完全二叉树，即除最后一层外的其余层都是满的，并且最后一层要么是满的，要么在右边缺少连续若干节点。
    具有 n 个节点的完全二叉树的深度为 log2(n) + 1，深度为 k 的完全二叉树至少右 2^(k-1) 的节点，至多有 2^k - 1 个节点。

  • 父节点的值不小于或不大于子节点的值

堆排序就是合理使用堆的特性，即堆顶元素一定是最大（或最小）值，通过将该值暂存后，对剩余元素重复进行堆化的操作达到排序的效果。

时间复杂度：
设有 n 个元素的待排序序列，其中构建堆结构的时候，需要从最后一个元素的父节点开始，向上构建堆，最后一个元素的父节点为 [(n - 1 - 1)/2]，因此外层循环需要执行 O((n-2)/2)，内层循环为一个递归函数，执行时间为 2^k >= n --> k >= log2(n) ~ O(log2(n)) ，元素交换及已排序的元素确认的执行时间近似为 O((n-1)*log2(n))。

因此总的时间复杂度可以近似为：O(n*log2(n))

空间复杂度：
因为是原地排序，所以空间复杂度为 O(1)。

归并排序

将两个有序子序列合并成一个有序序列的过程的叫做归并，又称二路归并，而实际的情况下，大部分都是多路归并，即存在两个及以上的有序子序列的合并操作。

在这种多路归并的常见的情况，一般是是两两做一个二路归并，然后对合并后缩小一般的有序序列再次进行两两归并，直到只剩一个有序序列位置。

实现逻辑有两种：

  • 自底向上
    使用迭代的方式进行归并操作，第一次将每一个元素作为一个有序子序列，进行两两归并，第二次将第一次的结果，即有两个元素的有序序列进行两两归并，….直到最后一次只有一个序列为止。

  • 自顶向上
    使用递归的方式进行归并操作，将整个序列分为两个，对两个子序列再进行两个子序列的拆分，重复执行，直到每一个有序序列为一个元素为止，然后同样进行自底向上一样的操作。

时间复杂度：
设有 n 个元素的待排序序列，外层循环的时间复杂度为 2^k >= n --> k >= log2(n) ~ O(log2(n))，内层的二路归并，因为每一个元素都需要遍历，所以执行的时间复杂度为 O(n)，所以总的时间复杂度为 O(n*log2(n))。

空间复杂度：
因为是需要一个辅助序列，并且辅助序列的长度等于资源序列的长度，所以空间复杂度为 O(n)。

计数排序

计数排序（Counting Sort），借助指定索引的容器，将元素按值索引匹配结果放入对应的容器中，实现收容，然后遍历容器结果，按顺序将元素一一输出，因为需要考虑排序算法的稳定性，所以排序时时按容器的收容顺序反向输出。

计数排序的辅助容器大小受排序序列的最大最小值所影响，若以最大值为标准构建容器，若值很大，则会造成大量空间的浪费，因此容器的容量优化为了 k = 最大值 - 最小值 + 1，与原来的相对来说有了一定程度的优化，但若差值很大，而元素很少，依旧浪费空间。

时间复杂度：
设有 n 个元素的待排序序列，构建容器集合需要找到待排序序列的最大值与最小值，执行的时间复杂度为 O(n)，然后，遍历待排序序列，将元素放在指定的索引位置，执行的时间复杂度为 O(n)，容器遍历，设容器的大小为 k，那么外层循环的次数为 O(k)，因此总的时间复杂度为 O(2n + k) ~ O(n + k)。

空间复杂度：
因为是需要一个辅助序列，并且辅助序列的长度等于 k = max - min + 1，所以空间复杂度为 O(k)。

稳定性：
设存在序列：[1, 2, 3, 4, 5, 6, 2', 3', 4']，以待排序的元素作为辅助容器的索引，元素 2 的插入顺序为 [2, 2']，元素的输出顺序为，[1, null, 2', ...] -> [1, 2, 2', ...]。

其中元素的相对位置并无变化，所以是 稳定排序

桶排序

桶排序（Bucket Sort），借助指定索引的容器，将元素按值索引匹配结果放入对应的容器中，实现收容，在容器的每一个索引位上都是一个有序的子序列，最终遍历容器将有序子序列合并成一个有序序列。

桶排序的辅助容器大小受排序序列的最大最小值所影响，若以最大值为标准构建容器，若值很大，则会造成大量空间的浪费，因此容器的容量在计数排序的基础上更近一步优化为了 k = (最大值 - 最小值)/10 + 1。

需要注意，桶排序不是直接用数组作为辅助容器的，而是采用数组集合和链表集合的结合来作为辅助容器，数组集合有着遍历的优势，链表则有着插入删除的优势，同时元素插入时的匹配标准应该和容器的标准统一，即 index = (item - min)/10，计算的结果作为索引值。

时间复杂度：
设有 n 个元素的待排序序列，构建容器集合需要找到待排序序列的最大值与最小值，执行的时间复杂度为 O(n)，初始化容器的时间复杂度为 O(n)，然后，遍历待排序序列，将元素放在指定的索引位置，执行的时间复杂度为 O(n)，容器遍历，设容器的大小为 k，那么外层循环的次数为 O(k)，因此总的时间复杂度为 O(3n + k) ~ O(n + k)。

空间复杂度：
因为是需要一个辅助序列，并且辅助序列的长度等于 k = (max - min)/10 + 1，容器的结构采用了 数组 + 链表 的形式，数组的大小为k，而链表的大小为元素的个数，即 n，所以空间复杂度为 O(n + k)。

稳定性：
因为在容器内排序的时候使用的是插入排序的算法，所以是有序子序列是稳定的，那么总的有序序列也是稳定的，所以是 稳定排序

基数排序

基数排序（radix sort），以排序序列的元素的数值位数（个、十、百、…）作为排序标准，在辅助容器（或，桶）中，按基准将元素放在指定的容器索引（或，称之为子桶）内，若存在多个值，则在子容器中进行排序，最终将有序子序列合并成一个有序序列。

排序主要有两种算法：

  • 最高位优先
    Most Significant Digit first，简称MSD法：先按k1排序分组，同一组中记录，关键码k1相等，再对各组按k2排序分成子组，之后，对后面的关键码继续这样的排序分组，直到按最次位关键码kd对各子组排序后。再将各组连接起来，便得到一个有序序列。

  • 最低位优先
    Least Significant Digit first，简称LSD法：先从kd开始排序，再对kd-1进行排序，依次重复，直到对k1排序后便得到一个有序序列。

时间复杂度：
设有 n 个元素的待排序序列，构建容器集合需要找到待排序序列的最大值，执行的时间复杂度为 O(n)，初始化容器的时间复杂度为 O(10)，然后，遍历待排序序列，将元素放在指定的索引位置，执行的时间复杂度为 O(n)，外层循环执行时间复杂度为最大值的高位数 k，那么外层循环的次数为 O(k)，因此总的时间复杂度为 O(n*k + 10 + n) ~ O(n*k)。

空间复杂度：
因为是需要一个辅助序列，辅助序列的长度为位数的取值范围，即数组的大小为 k = 10，而内层集合的大小为元素的个数，即 n，所以空间复杂度为 O(n + k)。

稳定性：
元素遍历进入容器中时，按照加入顺序重新会原序列，重复进行操作，元素的相对位置并无变化，所以是 稳定排序
