Spring 框架之初

发表于 2020-08-22 | 分类于 notebook

概述

秉着 “高内聚，低耦合” 的程序设计宗旨，开发框架层出不穷，其中使用广泛的框架之一便是 Spring 全家桶系列，它涵盖了几乎所有的主流程序设计方向，Web、分布式、微服务等等，本章节主要介绍在学习框架前需要了解的知识技术。

框架学习主要分为几个小章节：

  • Spring 框架介绍
  • 类加载机制和双亲委派模型
  • SPI 机制
  • IOC 容器

什么是 Spring

Spring 框架是 Java 平台的一个开源的全栈（Full-stack）应用程序框架和控制反转容器实现，一般被直接称为 Spring。该框架的一些核心功能理论上可用于任何 Java 应用，但 Spring 还为基于Java企业版平台构建的 Web 应用提供了大量的拓展支持。虽然 Spring 没有直接实现任何的编程模型，但它已经在 Java 社区中广为流行，基本上完全代替了企业级JavaBeans（EJB）模型。

轻量级的特性使得框架将核心层实与应用层的实现逻辑独立开来，核心层提供关键的对象容器管理实现，应用层包括 Web、AOP、JDBC、Transaction ……，每一个具体的模块都是一个独立可运行的应用。

官网：https://spring.io/projects/spring-framework

框架核心层提供了 org.springframework.beans 和 org.springframework.context 两个关键包，前者核心接口 BeanFactory，后者核心接口 ApplicationContext，这两个核心接口在IOC容器管理中起了非常重要的作用，之后会详细介绍。

类加载机制和双亲委派模型

了解类加载机制，在类加载过程中，加载器是如何工作的，加载过程又是怎样的。

类加载机制

通过一个类的全限定类名来获取描述此类的二进制文件流，在Java虚拟机外部程序中，实现加载此二进制字节流的过程称为类加载过程，该模块称之为类加载器。

只有被同一个类加载器加载的类才有可能相等，相同字节码，被不同类加载器加载得到的类是不相等的。

类加载过程：加载 -> 连接【验证 -> 准备 -> 解析】-> 初始化 -> 使用 -> 卸载

类加载划分：

  • Bootstrap ClassLoader
    启动类加载器，加载JDK：%JAVA_HOME%\lib 目录下的类
  • Extension ClassLoader
    扩展类加载器，加载JDK：%JAVA_HOME\lib\ext 目录下的类
  • Application ClassLoader
    应用程序类加载器，加载用户指定路径下的类
  • User ClassLoader
    自定义加载器，加载指定路径下的类

双亲委派模型

因为类加载器的种类较多，所以为了使各加载器之间能够协调工作，双亲委派模型便诞生了，他的核心加载过程便是将类加载的过程交由父类加载器进行加载，若父类加载器无法加载类时，才交由自己的类加载器进行加载，若都无法加载，则会抛出 ClassNotFoundException。

SPI机制

Service Provider Interface，简称 SPI，字面上翻译是服务提供接口API，是JDK内置的一种服务提供发现机制，用于运行时动态加载实现。

常用的JDBC等框架就是采用的这种策略模式，调用者可以根据实际使用需要，启用、扩展、或者替换框架的实现策略。

IOC 容器

IOC 容器的设计思想是实现对象创建的控制反转，将对象的创建从客户端转交给容器统一创建并管理，客户端需要用的时候，通过某些方式注入对象即可，常用的注入方式是注解或配置文件。

实现方式：

  • XML文件配置
  • 注解
  • 工厂模式
  • 其他配置方式

无论哪一种实现方式，最终目的都是想要实现对象的创建，和对象属性的注入，实现程序最大程度上的解耦。

具体实现方式及原理在这里不做详细的描述，后续会有一篇文章专门讲述它，跳转地址：Spring 对象管理。
