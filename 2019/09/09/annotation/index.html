注解的学习和使用

发表于 2019-09-09 | 分类于 notebook

序言

注解在Java编程中使用非常的广，使用频率也非常的高，但多数情况下对于注解，很多人都只是局限于会用，并不知道其原理，写这篇文章的目的就是做学习笔记，记录注解的学习过程。
Spring，程序员都听过的开源技术框架，学习其注解的使用场景，和实际效果。
Lombok，不要觉得这个东西没听过就不好用，接受它会让你的代码更加简洁，易于阅读。

目的

了解注解的定义过程，元注解的使用及注解生命周期的设定，学会自定义注解，同时学习主流框架中的常用注解，了解其注解的内部实现，学习其设计模式，了解其结构。

开始前

此技术框架以注解的方式使用在项目之中，因此，学习前需要了解关于 Java 注解的一些基础知识。

  • 元注解
  • 如何自定义注解

元注解

元注解的作用就是负责注解其他注解。在 java.lang.annotation 包下可以找到以下元注解：

  • @Target
  • @Retention
  • @Documented
  • @Inherited

@Target

@Retention

@Documented
Documented,，记录，文档，用于描述其它类型的annotation应该被作为被标注的程序成员的公共API，因此可以被例如javadoc此类的工具文档化。Documented是一个标记注解，没有成员。

@Inherited
Inherited 元注解是一个标记注解，@Inherited阐述了某个被标注的类型是被继承的。如果一个使用了@Inherited修饰的annotation类型被用于一个class，则这个annotation将被用于该class的子类。

自定以注解

了解了以上元注解之后，就可以开始学着自定以注解了，具体的实现流程是 public @interface 注解名 {定义体} ，其中自定义注解参数可支持数据类型：

  • 基本数据类型（int,float,boolean,byte,double,char,long,short)
  • String
  • Class
  • enum
  • Annotation
  • 以上所有类型的数组

需要注意的是：

  • 注解内部参数值只能是 public 或 Default修饰符所修饰
  • 参数类型必须符合可支持的类型

Lombok

开源的异端，使用的时候会觉得和以前的代码风格迥异，不符合其完整性，但使用后又觉得，这是一个非常不错的框架，能够提升开发效益，使用时必须结合相应插件，不然编辑器会自动检测异常。

@Data

使用场景：
在一个存在众多类对象的系统中，部分类型存在多个成员变量，一旦修改其中的某个值，就需要修改其所对应的 getter 和 setter 方法，这个修改的过程效率极低。Lombok的出现大大提升了开发效益，在创建实体，并构造其内部变量方法时，发挥巨大的效用。

@Data 作为使用最为广泛的注解之一，使用者觉得很方便，但查看源码以后，就会觉得一脸懵逼，在了解了自定义注解的关系之后，可以看出，@Data 注解的作用对象是类、接口（注解）、以及枚举类等，只存在于源文件中，及编译器有效，在编译之后会取消注释。
