集合类源码

发表于 2019-10-05 | 分类于 notebook

序言

作为使用最为广泛的工具类及接口之一，在程序开发中有着不可替代的地位，其中使用最多的有 List、Map、Set 及他们的派生类等，结合集合处理的工具类，可以实现数据的有序排列，去除重复等操作，本篇文章通过源码层面对这些集合接口（类）进行解析。

集合类型介绍

集合，顾名思义，就是相同类型的数据的一个集合体，使用集合的意义在于将一系列相同类型的数据放在同一空间下，有利于数据的排序和比较等操作，主要使用的集合有以下几种：

  • List
  • Map
  • Set

依据不同的使用场景使用不同的集合类型，相对的，在一定情景下结合使用会更有效，以集合的形式衍生出的工具类非常多，一些常用的类数据库就是采用的键值对的形式存储数据，所以学好集合，对技术学习提升很有帮助。

List

列表式集合，其主要实现有：

  • ArrayList
    数组结构集合，存储方式与数组类型，通过下标确定元素位置，方便获取。

  • LinkedList
    链式结构集合，通过链式结构，可以更灵活的实现元素的添加和删除。

ArrayList

数组结构集合，能够实现元素的存储和获取，通过下标的形式确定元素位置，主要特点有集合大小、集合元素类型及集合元素顺序等。

  • 优点
    数组式的存储，每一个元素都有唯一的下标，应该可以实现快速的获取。

  • 缺点
    插入元素的时候，因为元素都是确定了下标的，所以插入元素时需要先去查找下标位置，然后实现移动，在执行插入操作。

变量：默认容量、初始元素集合、集合大小

序列化：transient关键字
1）一旦变量被transient修饰，变量将不再是对象持久化的一部分，该变量内容在序列化后无法获得访问。

2）transient关键字只能修饰变量，而不能修饰方法和类。注意，本地变量是不能被transient关键字修饰的。变量如果是用户自定义类变量，则该类需要实现Serializable接口。

3）被transient关键字修饰的变量不再能被序列化，一个静态变量不管是否被transient修饰，均不能被序列化。

如上定义可以看出：

  • DEFAULT_CAPACITY
    列表默认的初始容量为10
  • size
    列表所包含的元素个数，默认的初始化大小为0
  • EMPTY_ELEMENTDATA
    在初始化阶段传入参数时使用
  • DEFAULTCAPACITY_EMPTY_ELEMENTDATA
    无参构成器初始化时使用默认容量的空元素容器，集合扩容时会使用到此变量
  • elementData
    存储ArrayList元素的数组缓冲区，ArrayList的容量是此数组缓冲区的长度，添加第一个元素时，
    任何具有 elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA 的空 ArrayList 都将扩展为 DEFAULT_CAPACITY

需要注意的点：

  • 容量（CAPACITY）和大小（SIZE）是两个概念，前者指的是当前集合所能容纳的元素量，后者为当前容器已经存储的元素量。
  • EMPTY_ELEMENTDATA 和 DEFAULTCAPACITY_EMPTY_ELEMENTDATA 的区别在于，前者用于有参构造器的初始化，后者用于无参构造器初始化。
    • 前者的目的就是和后者区分开来，在用户使用有参构造出容量为0的集合列表时，此时需要创建一个空的容器，如果创建多个实例，就会创建多个空的容器，因此使用 EMPTY_ELEMENTDATA 来指代一个空的容器对象，在需要用到的地方，将引用指向此对象即可。
    • 在扩容标准上二者也存在差异，在后面扩容内容上会详细介绍。

初始化：无参构造器，指定大小构造器，带元素初始化构造器，用于初始化集合列表

结合全局变量来看：

  • ArrayList()
    无参构造器，不传入参数直接初始化一个集合对象，内部实现为 this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA，直接对元素集合进行赋值（引用指向）为默认容量的空列表容器。
  • ArrayList(int initialCapacity)
    指定初始化容量，判断初始化容量的合理性，在这里使用到了0（当 initialCapacity = 0）容量的空列表容器对象 EMPTY_ELEMENTDATA。
  • ArrayList(Collection<? extends E> c)
    指定数据初始化，接收的参数为集合 Collection 及其派生类对象，传入对象元素大小为0时，elementData 引用指向 EMPTY_ELEMENTDATA 对象。

需要注意的点：

  • 新创建的实例的初始容量为0，当添加第一个元素时，才会扩容为默认的容量 DEFAULT_CAPACITY = 10
  • 参数为集合类型时，存在对当前集合对象大小（size）赋值的过程。

常用操作：删除

通过索引删除：remove (int index)

  • rangeCheck(index)
    验证索引，索引越界异常

  • 删除原理
    对符合条件的索引，执行删除的指定位置的重新拷贝操作，将索引之后的元素全部左移，同时集合大小（size）缩小，将容器的末尾元素设置为 null

匹配对象删除：remove (Object o)

  • fastRemove(int index)
    匹配当前对象在集合中第一次出现的位置索引，删除原理同上，找到对象索引位置后进行复制操作。

常用操作：遍历

常用的有 for 及 foreach（增强for循环）等，迭代器（Iterator）等，依据不同的场景使用不同的循环可以提高开发的效率。

迭代循环：Iterator
通过 iterator() 方法将返回一个迭代器对象，此对象为 ArrayList 的内部类，实现了 Iterator 接口。

  • hasNext()
    返回 boolean 结果，当前游标值不为集合大小值时，一般结合 while 循环使用，作为限制条件

  • next()
    执行前会校验变更版本数，如果不等于预期的结果，会抛出并发修改异常
    游标向后移动，获取当前游标所对应的元素值

  • remove()
    调用的是 ArrayList.this.remove(lastRet)方法，也就是删除原理是通过索引取匹配元素并进行删除。

LinkedList

链式结构集合，双向链表模式可以实现正向及逆向的数据插入等，和数组结构的区别在于链表没有容积的概念，不需要考虑扩容所带来的问题。

  • 优点
    链式存储对于元素的新增非常高效，只需要知道插入元素的前一个节点就可以实现元素的插入。

  • 缺点
    获取元素没有数组结构方便。

结点 Node

  • item
    结点值
  • next
    当前结点的下一个结点
  • prev
    当前结点的前一个结点
  • Node(prev,element,next)
    构造结点，根据插入结点的位置对前后结点进行初始化

变量：集合大小、首尾节点指针

  • size
    集合大小，初始值为0
  • first
    内部类Node的对象实例，用于标识集合的首结点指针
  • last
    内部类Node的对象实例，用于标识集合的尾结点指针

初始化集合

和 ArrayList 不同的是，链式存储没有容积的概念，理论上只要内存允许，可以无限的插入并存储数据。初始过程分为两种，无参初始化，和附加初始数据的初始化，具体实现看插入数据的逻辑。

新增

单个结点的插入分为两种方式，addFirst（首结点前插入）和 addLast（尾结点后插入），默认的添加都是直接在尾结点之后插入数据，具体实现分为 linkFirst 和 linkLast，当链表只有一个结点是，保证首尾结点一致。

序列化

Java中对象的序列化指的是将对象转换成以字节序列的形式来表示，这些字节序列包含了对象的数据和信息，一个序列化后的对象可以被写到数据库或文件中，也可用于网络传输，一般当我们使用缓存cache（内存空间不够有可能会本地存储到硬盘）或远程调用rpc（网络传输）的时候，经常需要让我们的实体类实现Serializable接口，目的就是为了让其可序列化。

当然，序列化后的最终目的是为了反序列化，恢复成原先的Java对象，要不然序列化后干嘛呢，所以序列化后的字节序列都是可以恢复成Java对象的，这个过程就是反序列化。

位移符

  • >>
    右移符号，表示将当前数右移，正数高位补0，负数高位补1
    以一个字节整数为例-10 >> 1：
    -10 的原码为 10001010 第一位是符号位（1表示负数，0表示正数），其余位表示数值
    -10 的反码为11110101 符号位不动，其余位按位取反
    -10的补码=反码+1= 11110101 +1 = 11110110
    右移一位后所得：11111011 -> 反码：11111010 —> 原码：10000101 = -5
  • <<
    左移符号，表示将当前数左移，无论正负，低位补0
  • >>>
    表示无符号右移，也叫逻辑右移，即若该数为正，则高位补0，而若该数为负数，则右移后高位同样补0
